## What This Project Does

This project analyzes mobile app user behavior 
to understand how different user segments engage 
with product features. The goal is to help product 
teams make data-driven decisions about which 
features to prioritize and which user segments 
need better retention strategies.

## The Business Questions I Answered

1. Do female and male users have different 
   session lengths?
2. How do engagement rate, feature usage, 
   and retention scores compare across users?
3. Which behavioral metrics show the biggest 
   gap between user segments?

## Key Findings

- Female users show slightly higher average 
  session length than male users — suggesting 
  different feature preferences worth investigating
- Engagement Rate is the strongest performing 
  metric compared to Feature Usage Score 
  and Retention Score
- Retention Score is the weakest metric — 
  indicating the app has an engagement problem 
  worth addressing in the product roadmap

## Tech Stack

## Dashboard Overview

Sheet 1 — Session Length by Gender
Compares average session length between 
female and male users to identify behavioral 
differences in app usage patterns.

Sheet 2 — Behavioral Metrics Comparison
Compares three key product health metrics — 
Engagement Rate, Feature Usage Score, and 
Retention Score — to identify the weakest 
area for product improvement focus.

## What This Analysis Means for Product Teams

When retention score is lower than engagement 
rate, it signals users find the app interesting 
but aren't coming back consistently. This is 
a classic product problem — good acquisition, 
weak habit formation. A product analyst would 
use this finding to recommend push notification 
improvements, onboarding flow changes, or 
feature discovery optimization.
